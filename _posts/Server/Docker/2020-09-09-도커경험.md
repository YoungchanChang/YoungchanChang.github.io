# 도커를 쓰면 좋았던 이유

### 서버 관리

- 도커를 쓰면 소프트웨어 패키지가 일괄적으로 관리되서 서버 관리가 쉬워진다.

- 나는 python 3.5에서 실행시키고, 다른 사람은 python 3.6에서 코드를 실행시켰다.


### 다른 개발 환경

- 윈도우 환경에서도 똑같이 테스트 할 수 있다.

- 다른 개발자는 MAC에서 실행하고, 나는 윈도우에서 코드를 작성했다. 그런데 같은 코드라도 경로 설정 등이 달라서 실행이 안 되었다.
